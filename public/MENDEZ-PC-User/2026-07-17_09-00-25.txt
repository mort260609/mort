  # Zapper: Add WETH as an Alt Entry Token
  
  Add a WETH-deposit path to the zapper, mirroring the existing
  `zapDepositUsdc` but reversed: user provides WETH only, the contract
  swaps the portion needed to cover the USDC leg, then deposits into
  `NexaiCore` same as before.
  
  ## Approach
  
  Add `zapDepositWeth(uint256 nexaiAmount, uint256 maxWethIn)` to
  `NexaiZapper.sol`, following the same structure as `zapDepositUsdc`:
  
  1. Read `wethWeight`/`usdcWeight` from Core, same
     `_basketIsExactlyTwoTokens()` guard already in place
  2. Compute `requiredWeth` and `requiredUsdc` via the same round-up
     formula (duplicated from Core, same as the existing function —
     keep the comment flagging this needs to stay in sync)
  3. `maxWethIn` is the slippage guard — total WETH the user is willing
     to spend across the direct WETH leg + the swap leg (mirrors
     `maxUsdcIn`'s role in the existing function)
  4. Pull `maxWethIn` WETH from the caller upfront
  5. Swap the budget remaining after the direct WETH leg
     (`maxWethIn - requiredWeth`) into exactly `requiredUsdc`, via
     `exactOutputSingle` with `tokenIn: WETH, tokenOut: USDC`
  6. Approve Core for the exact WETH/USDC amounts, call `deposit()`
  7. Forward the minted NEXAI to the real caller (same pattern —
     Zapper is `msg.sender` from Core's perspective)
  8. Refund unspent WETH, reset approvals to zero, same
     zero-custody-between-transactions guarantee as the existing function
  
  ## Tests
  
  Mirror the existing `NexaiZapper.t.sol` coverage for the new function:
  - Correctness (full deposit + refund + zero-dust invariant)
  - Reverts on excess slippage
  - Reverts when `maxWethIn` too low to cover the direct WETH leg
  - Reverts when basket isn't WETH/USDC
  - Reverts when basket has a third token
  
  ## Notes
  - Do not touch `zapDepositUsdc` — this is purely additive
  - Same testnet-only scope as everything else; no mainnet
  - USDT support is explicitly a separate follow-up, not part of this
    task — don't start on it

  zapDepositWeth is in NexaiZapper.sol — a mirror of zapDepositUsdc with the legs reversed: WETH is the direct leg, the swap buys requiredUsdc using WETH (exactOutputSingle with tokenIn: weth, tokenOut: usdc). Same round-up formula duplication (flagged with the same "must stay in sync with Core" comment), same zero-custody/refund/approval-reset pattern, same _basketIsExactlyTwoTokens() guard reused as-is. zapDepositUsdc itself — untouched.
  
  One thing had to change beyond pure addition, worth flagging: the test file's MockSwapRouter only knew how to price one direction (USDC-in/WETH-out). Calling it for the new WETH-in/USDC-out leg with the same formula would've silently computed the wrong amountIn and made the WETH tests pass or fail for the wrong reasons. I made it direction-aware (checks tokenIn/tokenOut against known weth/usdc addresses, applies the inverse formula for the reverse direction) so both legs price correctly off the same fixed rate. Verified by hand: at the seeded 2000 USDC/WETH rate, 2 NEXAI needs 1 WETH direct + 1 WETH via swap (buying 2000 USDC) = 2 ETH total — the numbers in test_ZapDepositWeth_Correctness reflect that.
  
  Added 6 new tests mirroring the USDC suite: correctness + zero-dust invariant, slippage revert, budget-too-low-for-direct-leg revert, and both basket-mismatch reverts (wrong token, third token present).
  
  Same caveat as before: I can't run forge build/forge test in this sandbox, so this is unverified until you run it locally — worth prioritizing the WETH tests specifically since the mock router change is new and unverified.

  // SPDX-License-Identifier: MIT
  pragma solidity ^0.8.20;
  
  import {IERC20} from "@openzeppelin/contracts/token/ERC20/IERC20.sol";
  import {SafeERC20} from "@openzeppelin/contracts/token/ERC20/utils/SafeERC20.sol";
  import {ReentrancyGuard} from "@openzeppelin/contracts/security/ReentrancyGuard.sol";
  import {NexaiCore} from "./NexaiCore.sol";
  
  /// @dev Minimal slice of Uniswap SwapRouter02's ABI (`IV3SwapRouter`) - only the
  /// exact-output single-hop swap this zapper needs. Note SwapRouter02 omits the
  /// `deadline` field the original SwapRouter has; callers rely on the tx itself
  /// landing or failing rather than a per-call deadline.
  interface ISwapRouter02 {
      struct ExactOutputSingleParams {
          address tokenIn;
          address tokenOut;
          uint24 fee;
          address recipient;
          uint256 amountOut;
          uint256 amountInMaximum;
          uint160 sqrtPriceLimitX96;
      }
  
      function exactOutputSingle(ExactOutputSingleParams calldata params) external payable returns (uint256 amountIn);
  }
  
  /**
   * @title NexaiZapper
   * @dev One-click single-token deposit into NexaiCore, entering via either USDC
   * (`zapDepositUsdc`) or WETH (`zapDepositWeth`). Swaps the portion of the deposit
   * needed to cover the other basket leg via Uniswap V3, keeps the rest as the direct
   * leg, then deposits and forwards the minted NEXAI to the caller - all in one
   * transaction. Scoped to the current two-token (WETH/USDC) basket; does not
   * generalize to arbitrary baskets. Holds no funds or state between transactions.
   */
  contract NexaiZapper is ReentrancyGuard {
      using SafeERC20 for IERC20;
  
      ISwapRouter02 public immutable swapRouter;
      NexaiCore public immutable core;
      IERC20 public immutable weth;
      IERC20 public immutable usdc;
      uint24 public immutable swapFee;
  
      event ZappedDeposit(address indexed user, uint256 nexaiAmount, uint256 usdcSpent);
      event ZappedDepositWeth(address indexed user, uint256 nexaiAmount, uint256 wethSpent);
  
      constructor(address _swapRouter, address _core, address _weth, address _usdc, uint24 _swapFee) {
          require(_swapRouter != address(0), "Invalid swap router address");
          require(_core != address(0), "Invalid core address");
          require(_weth != address(0), "Invalid WETH address");
          require(_usdc != address(0), "Invalid USDC address");
  
          swapRouter = ISwapRouter02(_swapRouter);
          core = NexaiCore(_core);
          weth = IERC20(_weth);
          usdc = IERC20(_usdc);
          swapFee = _swapFee;
      }
  
      /**
       * @dev Deposits USDC only and mints NEXAI to the caller.
       * @param nexaiAmount Amount of NEXAI to mint (18 decimals).
       * @param maxUsdcIn Upper bound on total USDC spent (direct leg + swap leg combined).
       * Acts as the slippage guard - reverts if the swap would need more than this covers.
       * @return usdcSpent The actual total USDC spent across both legs.
       */
      function zapDepositUsdc(uint256 nexaiAmount, uint256 maxUsdcIn) external nonReentrant returns (uint256 usdcSpent) {
          require(nexaiAmount > 0, "Amount must be greater than zero");
  
          uint256 wethWeight = core.basketWeights(address(weth));
          uint256 usdcWeight = core.basketWeights(address(usdc));
          require(
              wethWeight > 0 && usdcWeight > 0 && _basketIsExactlyTwoTokens(), "Basket is not WETH/USDC"
          );
  
          // Mirrors NexaiCore.deposit()'s own round-up formula so the amounts approved
          // below match exactly what deposit() will pull - see NexaiCore.sol.
          uint256 requiredWeth = (nexaiAmount * wethWeight + 1e18 - 1) / 1e18;
          uint256 requiredUsdc = (nexaiAmount * usdcWeight + 1e18 - 1) / 1e18;
          require(maxUsdcIn > requiredUsdc, "maxUsdcIn too low to cover direct USDC leg");
  
          usdc.safeTransferFrom(msg.sender, address(this), maxUsdcIn);
  
          uint256 usdcBudgetForSwap = maxUsdcIn - requiredUsdc;
          usdc.forceApprove(address(swapRouter), usdcBudgetForSwap);
          uint256 usdcSpentOnSwap = swapRouter.exactOutputSingle(
              ISwapRouter02.ExactOutputSingleParams({
                  tokenIn: address(usdc),
                  tokenOut: address(weth),
                  fee: swapFee,
                  recipient: address(this),
                  amountOut: requiredWeth,
                  amountInMaximum: usdcBudgetForSwap,
                  sqrtPriceLimitX96: 0
              })
          );
          usdc.forceApprove(address(swapRouter), 0);
  
          weth.forceApprove(address(core), requiredWeth);
          usdc.forceApprove(address(core), requiredUsdc);
          core.deposit(nexaiAmount);
          weth.forceApprove(address(core), 0);
          usdc.forceApprove(address(core), 0);
  
          // core.deposit() minted NEXAI to address(this) (the Zapper is msg.sender from
          // Core's point of view) - forward it on to the actual depositor.
          IERC20(address(core.indexToken())).safeTransfer(msg.sender, nexaiAmount);
  
          usdcSpent = requiredUsdc + usdcSpentOnSwap;
          uint256 refund = maxUsdcIn - usdcSpent;
          if (refund > 0) usdc.safeTransfer(msg.sender, refund);
  
          emit ZappedDeposit(msg.sender, nexaiAmount, usdcSpent);
      }
  
      /**
       * @dev Deposits WETH only and mints NEXAI to the caller. Mirrors zapDepositUsdc,
       * with the direct/swap legs reversed - direct leg is WETH, swap leg buys the USDC
       * needed with WETH.
       * @param nexaiAmount Amount of NEXAI to mint (18 decimals).
       * @param maxWethIn Upper bound on total WETH spent (direct leg + swap leg combined).
       * Acts as the slippage guard - reverts if the swap would need more than this covers.
       * @return wethSpent The actual total WETH spent across both legs.
       */
      function zapDepositWeth(uint256 nexaiAmount, uint256 maxWethIn) external nonReentrant returns (uint256 wethSpent) {
          require(nexaiAmount > 0, "Amount must be greater than zero");
  
          uint256 wethWeight = core.basketWeights(address(weth));
          uint256 usdcWeight = core.basketWeights(address(usdc));
          require(
              wethWeight > 0 && usdcWeight > 0 && _basketIsExactlyTwoTokens(), "Basket is not WETH/USDC"
          );
  
          // Mirrors NexaiCore.deposit()'s own round-up formula so the amounts approved
          // below match exactly what deposit() will pull - see NexaiCore.sol.
          uint256 requiredWeth = (nexaiAmount * wethWeight + 1e18 - 1) / 1e18;
          uint256 requiredUsdc = (nexaiAmount * usdcWeight + 1e18 - 1) / 1e18;
          require(maxWethIn > requiredWeth, "maxWethIn too low to cover direct WETH leg");
  
          weth.safeTransferFrom(msg.sender, address(this), maxWethIn);
  
          uint256 wethBudgetForSwap = maxWethIn - requiredWeth;
          weth.forceApprove(address(swapRouter), wethBudgetForSwap);
          uint256 wethSpentOnSwap = swapRouter.exactOutputSingle(
              ISwapRouter02.ExactOutputSingleParams({
                  tokenIn: address(weth),
                  tokenOut: address(usdc),
                  fee: swapFee,
                  recipient: address(this),
                  amountOut: requiredUsdc,
                  amountInMaximum: wethBudgetForSwap,
                  sqrtPriceLimitX96: 0
              })
          );
          weth.forceApprove(address(swapRouter), 0);
  
          weth.forceApprove(address(core), requiredWeth);
          usdc.forceApprove(address(core), requiredUsdc);
          core.deposit(nexaiAmount);
          weth.forceApprove(address(core), 0);
          usdc.forceApprove(address(core), 0);
  
          // core.deposit() minted NEXAI to address(this) (the Zapper is msg.sender from
          // Core's point of view) - forward it on to the actual depositor.
          IERC20(address(core.indexToken())).safeTransfer(msg.sender, nexaiAmount);
  
          wethSpent = requiredWeth + wethSpentOnSwap;
          uint256 refund = maxWethIn - wethSpent;
          if (refund > 0) weth.safeTransfer(msg.sender, refund);
  
          emit ZappedDepositWeth(msg.sender, nexaiAmount, wethSpent);
      }
  
      /// @dev NexaiCore's `basketTokens` is a public array with no length getter (only
      /// indexed access is auto-generated), so we detect a third basket token by probing
      /// index 2: it reverts with an out-of-bounds panic if the basket has at most two
      /// tokens, and succeeds if a third one exists. Without this, a basket expanded to
      /// WETH + USDC + anything else would still pass the weight checks above, then fail
      /// deep inside core.deposit() with a confusing raw revert instead of a clear one.
      function _basketIsExactlyTwoTokens() private view returns (bool) {
          try core.basketTokens(2) returns (address) {
              return false;
          } catch {
              return true;
          }
      }
  }

  /// @dev Stands in for Uniswap's SwapRouter02 in unit tests: swaps at a fixed,
  /// configurable USDC-per-WETH rate instead of doing real AMM math, so these tests
  /// exercise the Zapper's own accounting (not Uniswap's), deterministically and
  /// without needing a fork. Direction-aware so it can price both the USDC->WETH leg
  /// (zapDepositUsdc) and the WETH->USDC leg (zapDepositWeth) off the same fixed rate.
  contract MockSwapRouter is ISwapRouter02 {
      using SafeERC20 for IERC20;
  
      address public immutable weth;
      address public immutable usdc;
      uint256 public usdcPerWeth; // USDC (6dp) per 1e18 WETH
  
      constructor(address _weth, address _usdc, uint256 _usdcPerWeth) {
          weth = _weth;
          usdc = _usdc;
          usdcPerWeth = _usdcPerWeth;
      }
  
      function exactOutputSingle(ExactOutputSingleParams calldata params)
          external
          payable
          override
          returns (uint256 amountIn)
      {
          if (params.tokenIn == usdc && params.tokenOut == weth) {
              amountIn = (params.amountOut * usdcPerWeth) / 1e18;
          } else if (params.tokenIn == weth && params.tokenOut == usdc) {
              amountIn = (params.amountOut * 1e18) / usdcPerWeth;
          } else {
              revert("Unsupported pair");
          }
  
          require(amountIn <= params.amountInMaximum, "Too much requested");
  
          IERC20(params.tokenIn).safeTransferFrom(msg.sender, address(this), amountIn);
          IERC20(params.tokenOut).safeTransfer(params.recipient, params.amountOut);
      }
  }
  
  contract NexaiZapperTest is Test {
      NexaiIndexToken public indexToken;
      NexaiCore public core;
      NexaiZapper public zapper;
      MockSwapRouter public router;
  
      MockERC20 public weth;
      MockERC20 public usdc;
  
      address public alice = address(0x111);
  
      uint256 public constant WETH_WEIGHT = 0.5 ether; // 0.5 WETH per 1e18 NEXAI
      uint256 public constant USDC_WEIGHT = 1000 * 10 ** 6; // 1000 USDC per 1e18 NEXAI
      uint256 public constant USDC_PER_WETH = 2000 * 10 ** 6; // swap price: 2000 USDC per 1 WETH
  
      function setUp() public {
          indexToken = new NexaiIndexToken();
          core = new NexaiCore(address(indexToken));
          indexToken.grantRole(indexToken.CORE_ROLE(), address(core));
  
          weth = new MockERC20("Wrapped Ether", "WETH", 18);
          usdc = new MockERC20("USD Coin", "USDC", 6);
  
          address[] memory tokens = new address[](2);
          tokens[0] = address(weth);
          tokens[1] = address(usdc);
          uint256[] memory weights = new uint256[](2);
          weights[0] = WETH_WEIGHT;
          weights[1] = USDC_WEIGHT;
          core.updateBasket(tokens, weights);
  
          router = new MockSwapRouter(address(weth), address(usdc), USDC_PER_WETH);
          // Fund the router with WETH and USDC liquidity so it can pay out swaps in
          // either direction (USDC->WETH for zapDepositUsdc, WETH->USDC for zapDepositWeth).
          weth.mint(address(router), 1_000 ether);
          usdc.mint(address(router), 1_000_000 * 10 ** 6);
  
          zapper = new NexaiZapper(address(router), address(core), address(weth), address(usdc), 3000);
  
          usdc.mint(alice, 10_000 * 10 ** 6);
          weth.mint(alice, 10 ether);
      }
  
      function test_ZapDepositUsdc_Correctness() public {
          uint256 nexaiAmount = 2 ether; // mint 2 NEXAI
          uint256 requiredWeth = 1 ether; // 2 * 0.5
          uint256 requiredUsdc = 2000 * 10 ** 6; // 2 * 1000
          uint256 swapCost = 2000 * 10 ** 6; // 1 WETH at 2000 USDC/WETH
          uint256 maxUsdcIn = 4_500 * 10 ** 6; // includes a 500 USDC buffer for refund
  
          vm.startPrank(alice);
          usdc.approve(address(zapper), maxUsdcIn);
          uint256 usdcSpent = zapper.zapDepositUsdc(nexaiAmount, maxUsdcIn);
          vm.stopPrank();
  
          assertEq(usdcSpent, requiredUsdc + swapCost, "usdcSpent mismatch");
          assertEq(indexToken.balanceOf(alice), nexaiAmount, "NEXAI not forwarded to depositor");
          assertEq(usdc.balanceOf(alice), 10_000 * 10 ** 6 - usdcSpent, "alice USDC balance wrong (bad refund?)");

  # Frontend: Add WETH as a Second Entry Token Option
  
  The zapper is now deployed with both `zapDepositUsdc` and
  `zapDepositWeth`, tested and confirmed working on-chain:
  `0xb94612E8D1F5140ABc0dA3609AbA1c86cdf5F283`
  
  Update the Deposit tab so users can choose their entry token (USDC or
  WETH) for the one-click zap flow, not just USDC.
  
  ## Requirements
  
  - Update `VITE_NEXAI_ZAPPER` in `.env` to the new address (old one is
    orphaned — still works for USDC but doesn't have `zapDepositWeth`)
  - Add a simple token selector to the "zap" deposit mode — e.g. a small
    toggle/tab for "Pay with USDC" / "Pay with WETH", defaulting to USDC
    (matches the original pitch, per Dmitry)
  - **Live cost estimate for the WETH path**: mirror the existing
    `useSimulateContract` quote pattern, but reversed — quote
    `tokenIn: WETH, tokenOut: USDC, amount: requiredUsdc` (the WETH-path
    swap buys USDC, opposite of the USDC-path swap which buys WETH).
    Display "~X WETH" instead of "~X USDC" when WETH is selected
  - **Approve + deposit flow mirrors the USDC path exactly, token-swapped**:
    approve WETH for `maxWethIn` (buffered estimate, same 5% buffer logic),
    call `zapDepositWeth(nexaiAmount, maxWethIn)`
  - Add a `usdcZapperAllowance`-equivalent read for WETH's allowance
    toward the zapper (needed to correctly show "Approve WETH" vs
    "Deposit & Mint" button state, same pattern as the existing USDC one)
  - Balance/allowance reads should stay in the existing batched
    `useReadContracts` call where possible (only the quote itself needs
    `useSimulateContract`, same constraint as before)
  - Keep the "Advanced: deposit exact WETH + USDC directly" option
    unchanged — this is additive to the zap flow only

  - USDT — explicitly a separate, later task, don't start on it
  - No changes to `NexaiZapper.sol` itself — it's already done and
