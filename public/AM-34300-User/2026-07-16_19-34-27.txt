  # 🧠 First UX Audit (Based on Screens)
  
  ## 1. Visual hierarchy
  
  Current issue:
  
  Everything has almost identical visual weight.
  
  Tables
  
  Filters
  
  Buttons
  
  Cards
  
  Charts
  
  Dialogs
  
  Headers
  
  ...all compete equally.
  
  First priority:
  
  Create clear information hierarchy.
  
  ---
  
  ## 2. Filters
  
  The filter area is extremely dense.
  
  Current problems:
  
  • large pills
  
  • inconsistent spacing
  
  • weak grouping
  
  • no visual priority
  
  Opportunity:
  
  Turn filters into a cleaner enterprise filtering experience similar to:
  
  Atlassian
  
  Linear
  
  PowerBI
  
  Notion databases
  
  ---
  
  ## 3. Tables
  
  Tables dominate almost every screen.
  
  Problems:
  
  - heavy borders
  - too much teal
  - insufficient row grouping
  - little whitespace
  - low scanability
  
  Opportunity:
  
  More subtle grid
  
  lighter dividers
  
  better typography
  
  zebra rows
  
  sticky states
  
  hover hierarchy
  
  ---
  
  ## 4. Typography
  
  Currently typography has only one hierarchy level.
  
  Need:
  
  Display
  
  Section
  
  Card
  
  Body
  
  Metadata
  
  Labels
  
  Table values
  
  ---
  
  ## 5. Color System
  
  Too much saturated turquoise.
  
  Opportunity:
  
  Use color intentionally.
  
  Reserve accent color for:
  
  active
  
  selected
  
  success
  
  links
  
  important actions
  
  Everything else becomes neutral.
  
  ---
  
  ## 6. Buttons
  
  Many button styles compete:
  
  Outlined
  
  Filled
  
  Elevated
  
  Rounded
  
  Different shadows
  
  Need one unified component system.
  
  ---
  
  ## 7. Charts
  
  Charts visually disconnected from tables.
  
  Opportunity:
  
  Same spacing
  
  Same typography
  
  Same cards
  
  Same interaction language
  
  ---
  
  ## 8. Dialogs
  
  Example:
  
  Columns dialog
  
  Needs:
  
  better spacing
  
  grouping
  
  modern switches
  
  less empty space
  
  ---
  
  ## 9. Cards
  
  Some screens use cards.
  
  Others don't.
  
  Need unified card language.
  
  ---
  
  ## 10. Icons
  
  Current icons vary.
  
  Would establish:
  
  single icon family
  
  consistent size
  
  consistent stroke
